核心算法

多智能体路径规划 + 交通管制

深度融合路径规划与交通管理，提供三种互补算法以适应不同场景。

CBS (基于冲突的搜索)

适合小范围、交通冲突极度敏感的规划场景，规划效果最优。
性能: 20个机器人约5秒。

改进的 LaCAM2

适合中等范围、业务相对固定的场景，规划效果随计算时间增加而优化。
性能: 300个机器人最快1秒。

CLrp (私有)

灵活性最好，适合中大业务范围、业务变动大的场景，支持增量计算。
性能: 300个机器人约1.5秒。

资源分配算法

基于时间窗的灵活分配，采用基于优先级和时间的最佳匹配算法。应用于充电、工作站和存储库位分配。
性能: 分配1000个任务约1秒。

Core Algorithms

MAPF + Traffic Control

A deep integration of pathfinding and traffic management, offering three complementary algorithms for different scenarios.

CBS (Conflict-Based Search)

Ideal for small-scale, conflict-sensitive planning scenarios with optimal solutions.
Performance: ~5 seconds for 20 robots.

Improved LaCAM2

Suitable for medium-scale, relatively fixed business scenarios. Quality improves with computation time.
Performance: 300 robots in ~1 second.

CLrp (Proprietary)

Most flexible, ideal for large-scale, highly dynamic scenarios with incremental computing.
Performance: 300 robots in ~1.5 seconds.

Resource Allocation Algorithm

Flexible time-window based allocation using a best-match algorithm based on priority and time. Applied for charging, workstation, and storage location assignments.
Performance: Allocates 1000 tasks in ~1 second.

未来发展蓝图

支持分布式、高可用

系统按场景实例进行调度的业务模式天然支持分布式和高可用，借助成熟的分布式管理中间件，实现系统集群化。

预期优势： 提升系统容错能力和处理能力，支持更大规模的业务场景部署

支持更先进的智能设备

基于系统的设备管理和指令抽象模式，接入人形机器人等具身智能系统，实现对各类智能设备的统一管理和编排。

预期优势： 扩展系统应用范围，从传统自动化设备延伸到智能化设备生态

基于AI的自动化业务流生成模式

系统的任务流编排协议本质类似于简化的编程语言，天然与大语言模型具有亲和性，方便大语言模型根据用户需求，自动编排生成、调整业务任务流程，同时借助系统的仿真模拟接口，自动化仿真测试和验证。

预期优势： 大幅降低系统配置和维护成本，实现真正的智能化自动化运营

Future Development Blueprint

Distributed & High Availability Support

The system's scenario-instance-based scheduling business model naturally supports distributed and high availability architectures. With mature distributed management middleware, we will achieve system clustering capabilities.

Expected Benefits: Enhanced system fault tolerance and processing capacity, supporting larger-scale business scenario deployments

Advanced Intelligent Device Support

Based on the system's device management and command abstraction model, we will integrate humanoid robots and other embodied intelligence systems, achieving unified management and orchestration of various intelligent devices.

Expected Benefits: Expanding system application scope from traditional automation devices to intelligent device ecosystems

AI-Powered Automated Business Flow Generation

The system's task flow orchestration protocol is essentially like a simplified programming language, naturally compatible with large language models. This facilitates LLMs to automatically generate and adjust business task flows based on user requirements, while leveraging the system's simulation interfaces for automated testing and validation.

Expected Benefits: Significantly reducing system configuration and maintenance costs, achieving truly intelligent automated operations
